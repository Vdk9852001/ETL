Extract

Parse the field mapping template and load all legacy source tables

E
Run Extract
Reads Product_Template.xlsx and all source CSVs
This step reads your field mapping template (S4_Table / S4_Field / S47_Table / S47_Field) and loads all uploaded legacy source tables into memory. No files are written to disk yet.

Transform — Field Mapping

Fill each S4 target table from legacy source columns using the field map and join rules

T
Run Field Transformation
Applies join rules from config/join_rules.json (e.g. MAKT joins MARA on MATNR)
Each S4 target table is filled by pulling columns from the correct legacy tables. Tables that need a JOIN (e.g. MAKT ↔ MARA) are merged automatically. Date fields (YYYYMMDD format) are auto-detected and reformatted.

Value Mapping

Apply value lookup tables and object-specific rules (WAERS, MLAST, CURTP, ENTITLED)

V
Run Value Mapping
Each column in the transformed output is matched against your ValueMapping.xlsx sheets (sheet name = SAP field name). Values are replaced using the SAP47_Value → S4_Value lookup. Object-specific rules (WAERS from BWKEY, MLAST=3, etc.) are applied from config/object_rules.json.
